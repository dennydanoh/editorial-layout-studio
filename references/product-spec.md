# Product Spec

这个参考文档用于帮助模型稳定复刻或改造“图文排版网页 / 文章转卡片工具”这一类产品。

来源说明：

- 参考站点：`https://www.hezouyukuai.top/`
- 访问日期：`2026-04-07`
- 说明：以下内容由页面结构与前端脚本行为整理而成，适合作为实现规格，不要求逐字逐像素复刻。

## Product Goal

把用户输入的一整篇文章拆解为一组适合社媒发布、知识分发或品牌展示的 3:4 竖版图文卡片：

- 第 1 张为封面
- 后续为正文分页
- 全流程在浏览器本地执行
- 用户可即时预览并导出 PNG

## Core Inputs

页面至少包含这 3 个主输入：

1. 标题
2. 导语 / 角标文案
3. 正文

建议规则：

- 标题显示限制约 18 个字符
- 导语可选；为空时允许自动生成阅读信息
- 正文支持粘贴整篇文章，段落之间最好允许空行

## Cover Logic

封面应包含：

- 主标题
- 一行辅助信息或角标文案
- 一小段 teaser / 摘要

建议行为：

- 如果标题为空，尝试从正文第一段推断标题
- 标题需做字号回退，避免超出 3 到 4 行
- teaser 可由正文前若干字符自动拼接并截断

## Body Pagination Logic

建议采用下面的稳定策略：

1. 先清洗正文：去掉多余回车和首尾空白
2. 优先按空行拆段
3. 如果段落太长，再按句号、问号、感叹号等切成更小块
4. 在 canvas 中基于字体测量逐字换行
5. 依据可用高度把内容分页

一套可参考的实现参数：

- 画布：`1242 x 1660`
- 比例：`3:4`
- 正文最大宽度：约 `1024`
- 正文字号：约 `56`
- 行高：约 `90`
- 段间距：约 `42`
- 正文顶部起点：约 `220`
- 正文底部边界：约 `1492`

## Background System

建议支持两类背景：

1. 纯色背景
2. 自定义背景图

补充能力：

- 预设几种常用浅色卡片底色
- 允许用户自选颜色
- 背景图支持上传、移除和文件名提示
- 允许调节覆盖层透明度，确保文字可读

## Preview And Export

页面应至少提供：

- 全部页面的预览网格
- 每一张卡片的单独下载按钮
- “下载全部 PNG”按钮

导出建议：

- 文件名来自标题
- 先做非法字符清洗
- 加上页码，例如 `title-01.png`

## Persistence

建议把以下内容保存在 `localStorage`：

- 标题
- 导语
- 正文
- 背景色
- 透明度
- 背景图 data URL

## UX Layout

推荐布局：

- 左侧：输入与控制区域
- 右侧：预览区域
- 移动端：纵向堆叠

## Deployment Preference

优先产出这些形态之一：

- 原生 `index.html + styles.css + app.js`
- 轻量前端框架项目，但必须能静态导出

默认不要依赖数据库、登录、服务端图片任务队列。
